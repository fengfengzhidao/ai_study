# AGENTS.md

## 项目说明

这是一个 AI 职场提效课程制作仓库

主要用于沉淀课程结构 录课 Markdown 逐字稿和阶段性素材

当前重点是在制作写作表达提效相关课程文档

## 文件命名约定

课程文件使用章节编号加标题命名

示例

```text
3-1.怎么把写作任务正确交给AI.md
3-2.怎么让AI先搭结构而不是直接乱写.md
```

已确认或录制完成的逐字稿可以使用 `.txt`

录课辅助文档优先使用 `.md`

## Markdown 课稿风格

Markdown 是给学习者看的讲义

不是完整逐字稿

写作时遵守这些规则

- 语言要像老师教学生
- 示例尽量贴近生活和真实工作
- 大的小节控制在 5 到 6 个左右
- 避免过度拆标题
- 避免连续堆叠代码块
- 不使用 `---` 作为分隔线
- 句子末尾尽量不加标点
- 内容要能直接复制到 Markdown 文件里使用

## 编辑注意事项

修改课程内容时

先确认这一节和前后小节的边界

避免多个小节重复讲同一个能力

例如

- `3-2` 只讲搭结构
- `3-3` 只讲按已有结构起草第一版
- `3-4` 只讲改稿指令和局部修改
- `3-7` 只讲去 AI 味和发布前检查

新增内容时优先沿用已有口吻

不要把讲义写成工具说明书

也不要写成过度正式的培训手册
