# 安装和配置 AI Agent

## 这节课的目标

学会把 AI Agent 装到自己的电脑里

让 AI 不只是停留在聊天窗口

而是可以进入你的真实工作环境

帮你读文件

改文件

跑命令

看日志

甚至参与一个完整任务

这一节我们主要看 3 件事

- Claude Code 怎么安装
- 怎么用 cc-switch 切换不同模型配置
- Codex 命令行和桌面版怎么用

这里先提醒一句

工具更新非常快

所以这节课的重点不是背某一条命令

而是理解安装和配置背后的逻辑

正式安装时

一定要以工具官网和最新文档为准

## 一 AI Agent 和普通聊天 AI 有什么区别

普通聊天 AI 更像一个顾问

你把问题发给它

它给你回答

它通常不知道你电脑里有什么文件

也不会自己去跑命令

AI Agent 不一样

它更像一个能进入工作现场的助手

比如你让它处理一个项目

它可以做这些事

- 读取项目文件
- 搜索代码或文档
- 修改文件
- 运行命令
- 查看报错
- 根据结果继续调整
- 帮你整理改动说明

所以从第八章开始

我们讲的 AI 工作流

就不只是怎么问 AI

而是怎么让 AI 参与真实任务

不过能力变强之后

责任也更重要

你要知道它能访问哪里

能改什么

能不能运行命令

哪些操作必须先经过你确认

## 二 安装 Claude Code

Claude Code 是 Anthropic 的命令行 AI Agent

它适合在终端里处理项目

比如读代码 改代码 跑测试 整理文档

官方现在提供多种安装方式

比如原生安装 Homebrew WinGet npm 等

如果你是 macOS Linux 或 WSL

常见安装方式是

```text
curl -fsSL https://claude.ai/install.sh | bash
```

如果你已经有 Node.js 环境

也可以用 npm 安装

```text
npm install -g @anthropic-ai/claude-code
```

安装完成后

通常可以在终端里输入

```text
claude
```

进入 Claude Code

第一次使用时

它会要求你登录或配置账号

这里要注意

很多同学卡住的不是命令

而是网络和账号

比如官网访问不稳定

或者 API 服务连不上

所以我们要继续讲下一步

怎么管理不同模型配置

## 三 用 cc-switch 切换模型配置

有些同学没有稳定的海外网络环境

或者想使用国内可访问的大模型服务

这时候就会遇到一个问题

Claude Code 默认走自己的官方配置

但你可能希望切换到另一个 API 地址

比如某个国内模型服务商提供的兼容接口

这类场景可以用 cc-switch 这类工具来管理配置

它的作用不是替你提供模型

而是帮你管理不同 Claude Code 配置

比如

- 官方 Claude 配置
- 国内模型服务商配置
- 测试用配置
- 临时空配置

常见安装方式是

```text
npm install -g @hobeeliu/cc-switch
```

第一次可以初始化配置

```text
cc-switch init
```

然后按提示填写

```text
API Token
Base URL
模型相关配置
```

如果你以后有多个配置

可以查看配置列表

```text
cc-switch list
```

也可以切换到某个配置

```text
cc-switch use 配置名称
```

有些版本还支持切换后直接启动 Claude Code

```text
cc-switch use 配置名称 --launch
```

这里要讲清楚一个边界

cc-switch 只是配置切换工具

不是模型本身

你选择的国内模型服务商

必须自己确认这些信息

- 是否支持 Claude Code 需要的接口格式
- Base URL 怎么填
- API Key 从哪里获取
- 模型名称怎么写
- 价格和额度怎么算
- 数据是否会被用于训练
- 是否适合处理自己的项目资料

尤其是公司代码 客户资料 内部文档

不要随便丢给不清楚数据政策的服务

能跑起来只是第一步

能不能放心用

还要看安全和合规

## 四 安装 Codex 命令行

Codex 是 OpenAI 的 AI Coding Agent

命令行版本适合喜欢在终端里工作的同学

它可以在你选定的目录里

读取代码

修改文件

运行命令

帮你完成开发或文档任务

官方推荐的 npm 安装方式是

```text
npm i -g @openai/codex
```

安装完成后

进入你的项目目录

运行

```text
codex
```

第一次运行时

它会提示你登录

可以使用 ChatGPT 账号

也可以根据自己的情况使用 API Key

如果要升级到最新版

可以运行

```text
npm i -g @openai/codex@latest
```

如果你是 Windows 用户

可以直接在 PowerShell 里运行

有些项目如果需要 Linux 环境

也可以用 WSL2

这取决于你的项目本身

第一次使用 Codex 命令行

不要一上来就让它大改项目

可以先问一个低风险问题

```text
请先阅读这个项目
告诉我它主要是做什么的
不要修改文件
```

或者

```text
请帮我找出这个项目的启动方式
先不要运行命令
```

先让它读

再让它解释

最后再让它动手

这是使用 Agent 很重要的节奏

## 五 安装和使用 Codex 桌面版

Codex 桌面版更适合想要可视化管理任务的人

如果说命令行适合单线程深度操作

桌面版更像一个 Agent 工作台

你可以在里面同时管理多个任务

查看每个线程做了什么

看它改了哪些文件

也可以更方便地切换项目

Codex 桌面版目前支持 macOS 和 Windows

安装方式一般是

去官方 Codex App 页面

下载对应系统版本

安装后登录 ChatGPT 账号或配置 API Key

然后选择一个项目文件夹

第一次使用时

建议先选一个测试项目

不要直接选特别重要的工作项目

因为你要先熟悉这些事情

- 它能看到哪些文件
- 它会不会改文件
- 它运行命令前会不会问你
- 怎么查看 diff
- 怎么拒绝或接受改动
- 怎么停止一个任务

Codex 桌面版有几个很好用的地方

第一是多线程

你可以让不同 Agent 做不同任务

比如一个整理文档

一个检查代码

一个尝试修 bug

第二是内置 worktree 支持

多个 Agent 可以在隔离的工作副本里改

不容易互相打架

第三是更方便 review

你可以看它改了哪些文件

哪一处是新增

哪一处是删除

再决定要不要接受

第四是它不只适合写代码

也可以用来整理文档

处理表格

生成图片

使用技能

甚至做一些带浏览器或桌面应用的任务

所以这一节你要记住

Codex 桌面版不是简单的聊天窗口

它更像一个管理 AI Agent 的工作台

## 六 这一节先建立一个安全使用习惯

安装 AI Agent 之后

最重要的不是马上让它帮你干大活

而是先养成安全使用习惯

可以先记住这 5 条

```text
先读 不要急着改
先小任务 不要直接大项目
先看 diff 再接受
涉及删除 部署 密钥 数据库 一定谨慎
不确定时 让 Agent 先解释计划
```

第一次练习可以这样做

```text
请先阅读当前项目

告诉我
1 这个项目是做什么的
2 主要目录分别有什么作用
3 如果我要开始学习这个项目 应该先看哪些文件
4 请不要修改任何文件
5 请不要运行命令
```

等你熟悉之后

再逐步让它做更复杂的任务

比如

- 整理 README
- 查找某个函数在哪里用到
- 解释一个报错
- 写一个小工具脚本
- 给某个模块补注释
- 检查文档有没有错别字

最后记住一句话

AI Agent 的价值

不是让你完全放手

而是让 AI 真正进入你的工作现场

你给目标

它执行过程

你看计划 看权限 看结果

这样才是更安全 也更稳定的个人 AI 工作流
